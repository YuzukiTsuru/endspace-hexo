## 游戏与网页的交汇

游戏 UI 和网页 UI 的核心目标是一致的：**在有限的空间内高效传递信息，并引导用户完成目标操作**。但游戏 UI 往往走得更远——它需要同时服务于叙事和功能。

## 可借鉴的设计模式

### 状态可视化

游戏擅长用视觉元素表达抽象状态。《死亡搁浅》的货物管理界面用物理堆叠的方式展示物品，《赛博朋克 2077》用故障效果暗示系统异常。

网页中可以借鉴这种思路：

- 用**进度条**代替纯数字百分比
- 用**颜色渐变**表达状态变化（正常 → 警告 → 危险）
- 用**微动画**传递反馈（加载、成功、失败）

### 信息密度控制

《明日方舟》的作战界面是一个绝佳范例：在手机屏幕上同时展示地图、干员状态、技能冷却和敌人信息，却不会让玩家感到混乱。

关键技巧是**分层呈现**：

1. **核心层**：始终可见的当前行动信息
2. **上下文层**：悬停或聚焦时展示的补充信息
3. **细节层**：点击展开的完整数据

### 过渡与仪式感

游戏中的场景转换往往精心设计了过渡动画，让玩家在心理上做好准备。《明日方舟：终末地》的加载序列就是典型——它不是冰冷的等待，而是一段"系统启动"的仪式。

网页中 PJAX 过渡、骨架屏和加载动画都可以服务于同样的目的：**让等待成为体验的一部分，而非体验的中断**。

### 声音与触觉

游戏 UI 大量运用音效和震动来强化交互反馈。虽然网页对声音的支持有限，但：

- 微妙的 hover 音效可以增强按钮的存在感
- 振动 API（`navigator.vibrate()`）在移动端可以补充视觉反馈
- 过渡动画的缓动函数可以模拟物理质感

## 反面教材

游戏 UI 也有不适合网页的地方：

- **过度的装饰性元素**：游戏 UI 可以为了氛围牺牲效率，网页不行
- **非标准交互模式**：玩家愿意学习新的操作方式，网页用户不会
- **信息隐藏**：游戏可以有意隐藏信息制造探索感，网页应该让信息触手可及

## 总结

从游戏中学习网页设计，学的是**思维方式**而非具体手法。当你遇到一个设计问题时，问问自己：如果是游戏设计师，他们会怎么解决？
